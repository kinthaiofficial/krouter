# spec/07-auto-config.md ‚Äî Auto-Config & Install Wizard

**Modules**: `cmd/krouter-installer` + `internal/install` + `internal/config`
**Goal**: Get user from "downloaded .dmg" to "OpenClaw routing works" with zero technical operations.

---

## 1. Install wizard (first GUI launch)

5 steps. User makes decisions, GUI executes everything.

### Step 1: Language
Choose en / zh-CN. Saved to `~/.kinthai/settings.json`.

### Step 2: Permission preview
Inform user what will be installed (no admin password needed):
- daemon binary ‚Üí `~/.local/bin/` (macOS/Linux) or `%LOCALAPPDATA%\kinthai\` (Windows)
- system service registration (LaunchAgent / systemd --user / Task Scheduler)
- shell integration (~/.zshrc on macOS/Linux, env vars on Windows)

### Step 3: Execute install (automatic)
Progress bar showing each substep with checkmarks.

### Step 4: Detect & connect agents
Scan for installed agents:
- OpenClaw: ~/.openclaw/openclaw.json
- Hermes: ~/.hermes/ or ~/Library/Application Support/Hermes/
- Cursor: ~/.cursor/settings.json
- Claude Code: `claude` in PATH (`exec.LookPath`)

User checks which to enable. For checked items:
- OpenClaw/Hermes/Cursor: modify the agent's config file (with backup)
- Claude Code: ensure shell integration is in place

### Step 5: Completion
- Show connected agents
- For Claude Code (CLI): show "[Open new Terminal]" button
- Mark ~/.kinthai/installed so wizard doesn't re-run

## 2. Per-agent connection logic

### 2.1 OpenClaw

Path: `~/.openclaw/openclaw.json`

Modify these fields in `models.providers.anthropic`:
```json
{
  "baseUrl": "http://127.0.0.1:8402",
  "apiKey": "${ANTHROPIC_API_KEY}",
  "api": "anthropic-messages",
  "models": [...]                              // auto-detected via /v1/models
}
```

Process:
1. Read & parse existing config
2. Backup: write to `openclaw.json.kinthai-bak-{timestamp}`
3. Show user a diff preview (before / after)
4. Apply on user confirmation
5. If OpenClaw supports hot-reload via fsnotify, no restart needed
6. Otherwise, GUI offers "[Restart OpenClaw]" button (we kill PID + relaunch)

### 2.2 Hermes

Similar to OpenClaw. Path: `~/.hermes/config.toml`. Edit TOML, backup, diff, apply.

### 2.3 Cursor

Path: `~/.cursor/settings.json` (or platform equivalent). Edit:
```json
{
  "cursor.anthropic.baseUrl": "http://127.0.0.1:8402",
  "cursor.openai.baseUrl": "http://127.0.0.1:8402/v1"
}
```
Restart Cursor needed. GUI offers button to do so.

### 2.4 Claude Code (shell integration)

This is the only path that uses shell integration (because Claude Code reads
ANTHROPIC_BASE_URL from env).

**macOS / Linux** ‚Äî install wizard appends to user's shell rc:

```bash
# >>> kinthai router shell integration >>>
# Auto-generated by kinthai router install wizard. Do not edit manually.
eval "$(krouter shell-init)"
# <<< kinthai router shell integration <<<
```

Append to:
- bash: `~/.bash_profile`
- zsh: `~/.zshrc`
- fish: `~/.config/fish/config.fish`

Detection via `$SHELL` env var.

Marker comments allow precise removal during uninstall (sed between markers).

**Windows** ‚Äî set user environment variables (no shell rc concept):

```powershell
[Environment]::SetEnvironmentVariable(
  'ANTHROPIC_BASE_URL',
  'http://localhost:8402',
  'User'
)
[Environment]::SetEnvironmentVariable(
  'OPENAI_BASE_URL',
  'http://localhost:8402/v1',
  'User'
)
```

This writes to `HKCU\Environment`. All NEW processes inherit. Existing terminals
need to be restarted.

After install, GUI offers "[Open new Terminal]" button:
- macOS: detect `TERM_PROGRAM`, prefer iTerm/Warp, fallback Terminal.app
- Linux: `$TERMINAL` or `xdg-mime`
- Windows: Windows Terminal (`wt`), fallback PowerShell

## 3. Disconnect (per-agent)

Reverse of connection:
- OpenClaw/Hermes/Cursor: restore from `.kinthai-bak` OR precisely remove the 3 kinthai-added fields
- Claude Code: remove marker-wrapped block from shell rc (or remove env var on Windows)

User notified that disconnect is in effect for new sessions (shell-based agents).

---

## 4. Uninstall

Complete removal sequence:

1. For each connected agent: Disconnect (see ¬ß3)
2. Stop daemon: `launchctl unload` / `systemctl --user stop` / `schtasks /end`
3. Remove service registration: delete plist / unit file / scheduled task
4. Remove binaries: `~/.local/bin/krouter*` (or `%LOCALAPPDATA%`)
5. Windows: remove user env vars
6. Ask user: delete SQLite data and logs? (default: keep)
7. macOS: prompt user to drag `krouter.app` to Trash manually

GUI offers a single "Uninstall" button in Settings. CLI offers `krouter uninstall`.

## 5. Detection

```go
func DetectInstalledAgents() []Agent {
    var found []Agent
    
    if exists(home() + "/.openclaw/openclaw.json") {
        found = append(found, Agent{Name: "openclaw", Path: ...})
    }
    if exists(home() + "/.hermes/config.toml") {
        found = append(found, Agent{Name: "hermes", Path: ...})
    }
    if exists(home() + "/.cursor/settings.json") {
        found = append(found, Agent{Name: "cursor", Path: ...})
    }
    if path, err := exec.LookPath("claude"); err == nil {
        found = append(found, Agent{Name: "claude-code", Path: path})
    }
    
    return found
}
```

Run on install wizard step 4, also accessible later via "Settings ‚Üí Re-detect".

## 6. Backup strategy

Every config file modification creates a backup:
```
openclaw.json.kinthai-bak-2026-05-12-14-23-01
```

Backups never auto-deleted. User can manually clean. Restore = swap files.
